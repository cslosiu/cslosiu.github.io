<p>再探了一下 supabase.</p><p>之前我想好了新app的 data tables. 當然只是草稿了 table name 和相關欄位名. 面對 supabase 介面, 我當然不想再用 GUI 慢慢建立數據表. 我看到了它內置的 AI 命令介面: 右上角綠色菱形.&nbsp;</p><h2 style="text-align: left;">AI</h2><p>那我就說</p><blockquote></blockquote><blockquote><blockquote></blockquote></blockquote><blockquote><span style="font-family: courier;">follow tables please create them in effortapp schema: i will list the table name, followed by a colon, then fields. you try to define the type for them. and also you can see the FK between them.&nbsp;&nbsp;</span></blockquote><blockquote><span style="font-family: courier;">user: id, email, name<br />team: id, name&nbsp;<br />project: id, name, team id, start date, end date, ....</span></blockquote><blockquote></blockquote><blockquote></blockquote><p style="text-align: left;"></p><p><br />然後它就生成了相關的 create table SQL, 一鍵完成. 看了一下, 連 foreign key 也是正確的. 當然我用的約定俗成的命名慣例應該有幫助.</p><p>我再來:</p><blockquote><span style="font-family: courier;">also create two many-to-many tables to link up relevant tables.<br /></span><p style="text-align: left;"><span style="font-family: courier;">user_in_team: link up user and team&nbsp;</span>&nbsp;</p></blockquote><blockquote><p style="text-align: left;"><span style="font-family: courier;">task_by_user: link up task and user.</span></p></blockquote><p style="text-align: left;"></p><p>當然也完美運行, 2 個多對多的關係表也完成了. 甚至連 on delete cascade 也寫好.</p><p>我還打開 entity diagram 看了一下, 應該沒問題.</p><h2 style="text-align: left;">Database Functions</h2><p>然後我看到 supabase function. 也許是相當於 oracle 的 stored procedures. 問了AI確認, 我就叫它為幾個表都寫上 CRUD:</p><blockquote><p><span style="font-family: courier;">create CRUD functions for user, team....&nbsp; tables. execute the code directly.</span></p></blockquote><p>似乎它不會立即運行, 我還是要手動按一下運行鍵. 一下子20個 function 就出現了.&nbsp;</p><h2 style="text-align: left;">API</h2><p>我想到, 這麼好的東西, 也許可以經 HTTP 使用吧, 那豈不美妙?</p><p>當然是可以的:</p><p><span style="font-family: courier;"></span></p><blockquote><p><span style="font-family: courier;">POST https://&lt;your-supabase-url&gt;/rest/v1/rpc/create_user<br />Authorization: Bearer &lt;your-jwt-token&gt;<br />Content-Type: application/json</span></p><p><span style="font-family: courier;">{<br />&nbsp; "p_email": "user@example.com",<br />&nbsp; "p_name": "User Name"<br />}</span></p></blockquote><p><span style="font-family: courier;"></span></p><p>好吧, 似乎連 Vercel 都可以省去了...</p><p><br /></p>
