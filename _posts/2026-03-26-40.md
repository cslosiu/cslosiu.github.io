<p>實戰了幾輪, 付出真金白銀體驗, 有些體會:</p><p>DO:</p><p></p><ul style="text-align: left;"><li>最好先有一個 database schema, 同 AI 交待這個schema, 他會少犯錯.&nbsp;</li><ul><li>直接給他看 ER 圖也可以, 如 cloude 甚至注意到 ER 圖內的 relationship. 用圖的話, 要指定用哪一款數據庫. (mysql? oracle? prostgresql? ... )</li><li>不用圖, 給 CREATE TABLES 也可以</li><li>要設定好主鍵 PK 和外鍵 FK.</li></ul><li>在第一個提示就說好要求和細節. 就當你只有一次機會, 長一點沒關係.&nbsp;</li><ul><li>為了日後人手接力, 要交待: 平台為何 (如 Vercel), 有什麼數據庫 (如 Supabase), 它們用什麼方法連 (Supabase data API), 用的語言是什麼 (javascript), 要用什麼主要技術 (node js). 如果是 web app, 交待要用的 js 庫 (如 JQuery), CSS (如 bootstrap), template engine (如 Pug), 相關技術的 preference (如 HTMX).&nbsp;&nbsp;</li></ul><li>每次改動不要太多, 要限制他改的範圍.</li><li>及時TEST, 及時追加修正.</li><li>提醒AI要一並修改相似功能.</li><li>如DEBUG有阻礙, 盡量目測發現更多線索提供給AI.&nbsp;</li><li>處理有登入情況時, 提醒AI要為各功能加上token或cookie. 不然這些功能在加上LOGIN後可能會FAIL.</li></ul><div><br /></div><div>Do NOT:</div><div><br /></div><div>TBC</div><p></p>
